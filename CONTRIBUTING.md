# Contributing to Bronivik CRM

Спасибо за интерес к проекту! Мы рады любому вкладу.

## Как внести вклад

### Сообщение об ошибках

Если вы нашли ошибку:
1. Проверьте, не создан ли уже issue по этой проблеме
2. Создайте новый issue с подробным описанием:
   - Шаги для воспроизведения
   - Ожидаемое поведение
   - Фактическое поведение
   - Версия Go и ОС

### Предложение улучшений

1. Создайте issue с описанием предложения
2. Дождитесь обратной связи от maintainers
3. После согласования можно приступать к реализации

### Pull Requests

1. Fork репозитория
2. Создайте feature branch (`git checkout -b feature/amazing-feature`)
3. Внесите изменения
4. Запустите тесты (`make test`)
5. Запустите линтер (`make lint`)
6. Commit изменений (`git commit -m 'Add amazing feature'`)
7. Push в branch (`git push origin feature/amazing-feature`)
8. Создайте Pull Request

## Стиль кода

- Следуйте стандартным соглашениям Go
- Используйте `gofmt` для форматирования
- Добавляйте комментарии к экспортируемым функциям
- Пишите тесты для нового функционала

## Запуск тестов

```bash
# Все тесты
make test

# С покрытием
make test-coverage

# Линтинг
make lint
```

## Коммиты

Используйте понятные сообщения коммитов:
- `feat: добавлена функция X`
- `fix: исправлена ошибка Y`
- `docs: обновлена документация`
- `test: добавлены тесты для Z`
- `refactor: рефакторинг компонента A`

## Вопросы?

Если у вас есть вопросы, создайте issue с меткой "question".
